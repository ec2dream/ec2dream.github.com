Create an EC2Dream Environment

Core Concepts

Environment

This is the basic unit of administration in the EC2Dream System Admin. It identifies a group of servers and configuration identified by access keys, region and keypair. They are stored in a repository that is either local on your machine or another location that could be shared.

Access Key and Secret Access Key

The Access Key and Secret Access Key identify yourself when making requests to Amazon Web Services API. EC2Dream System Admin internally communicates with Amazon Web Services API to perform system admin tasks. You should never give out these keys to anyone.

Public and Private Key

Amazon also provides an X.509 Certificate consisting of a Public Key and Private Key to access Amazon Web Services API. Typically these are used on your server to access Amazon Web Services API. You can also supply your own X.509 certificates.

Keypair

This is a key to connect to your server running in Amazon EC2 via SSH. This is not to be confused with the X.509 Public and Private keys used to access the Amazon web Services API.

Creating an Environment

Run EC2Dream System Admin by executing the run.bat or run.sh.

1. The very first time EC2Dream is run it prompts for the location of the repository. Take the default local repository.



Then an initial screen is displayed:



2. Click on the icon , Select "Create New Environmnent" and press OK.



3. For Amazon enter a name for the environment, the access keys, and change the Region if your environment is in Europe.



NOTE: The access keys are available from http://aws.amazon.com under Your Account, Access Identifiers



4. For Eucalyptus select the Ecalyptus tab, enter a name for the environment, and enter the downloaded eucalyptus certificate zip file.



NOTE: The access keys are available from the 'Credentials' tab under you Eucalyptus account by pressing Show keys.



5. For OpenStack select the OpenStack tab, enter a name for the environment, the user and password , and the url. For trystack the url should be https://nova-api.trystack.org:5443. The credentials are under the API Option in the OpenStack console.




6. Press OK and the environment will be created. If you already have servers running or security groups defined you will see them in the tree view.





Creating a KeyPair

An SSH keypair is used for several purposes including connecting to Linux/OpenSolaris instances and retrieving your Windows Administrator password.
To generate a keypair, simply perform the following steps:

1. Click on the "KeyPairs" in the tree of EC2Dream System Admin.
2. Create a keypair by clicking on the new icon



3. Type in a name for your KeyPair, and click the "Create" button.



4. Copy the generated text including the -----BEGIN RSA PRIVATE KEY----- and -----END RSA PRIVATE KEY----- to a text file with a .pem suffix.
NOTE: For linux/Mac OSX clients remove permissions for all and group access

5. Make a backup copy of the file and store in a secure place as it cannot be repeatedly downloaded.

6. In the EC2Dream System Admin click on Settings in the tree view and in the Settings tab set the EC2_SSH_PRIVATE_KEY to the file created and then press "Save Settings".



7. PuTTy is the Windows SSH client software used to connect to the linux servers. It has its own keypair file format that can be generated from the keypair. If you are running EC2Dream on windows client press the "PuTTYgen Key Generator" button, press OK on the following screen



and you will see the following screen



8. In PuTTYgen press "Save Private Key" and then exit.
9. In the Settings tab set the PUTTY_PRIVATE_KEY to the file saved.



10. Save the settings.


Creating a Private and Public Key

The public and private key are used by the amazon command line interface to access the Amazon Web Services API.

1. For Amazon the public and private keys are X.509 Certificate access keys that can be created, and downloaded from http://aws.amazon.com under Your Account, Access Identifiers



2. For Eucalyptus the public and private keys are X.509 Certificate access keys that can be created, and downloaded from the 'Credentials' tab under you Eucalyptus account by pressing Download Certificate.
